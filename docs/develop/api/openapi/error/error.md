# openapi 错误

## 错误码处理

错误码分为两部分：

- http 状态码
- http body 返回的 json 中的 code

如：
![error](./error.png)

### http 状态码

| 值   | 含义                  |
| :--- | :-------------------- |
| 200  | 成功 |
| 204  | 成功，但是无包体，一般用于删除操作 |
| 201,202  | 异步操作成功，虽然说成功，但是会返回一个 error body，需要特殊处理 |
| 401  | 认证失败              |
| 404  | 未找到 API            |
| 405  | http method 不允许    |
| 429  | 频率限制              |
| 500  | 处理失败              |
| 504  | 处理失败              |

## 有关 traceID

在 openapi 的返回 http 头上，有一个 `X-Tps-trace-ID` 自定义头部，是平台的链路追踪 ID，如果开发者有无法自己定位的问题，需要找平台协助的时候，可以提取这个 ID，提交给平台方。

方便查询相关日志。
